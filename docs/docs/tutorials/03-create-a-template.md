# Tutorial 3: Create a Template

**Goal:** Create a new template with parameters and verify it renders correctly.

**Time:** ~15 minutes

**Who:** Project editors, project admins, org admins

---

## Prerequisites

You need the **project_editor** role or above on the target project.

---

## Step 1: Open the Templates List

Navigate to **Studio → Templates** in the sidebar.

<!-- SCREENSHOT: admin-templates-list -->

The templates list shows all templates across your accessible projects with their project, name, and last-modified date.

---

## Step 2: Create a New Template

Click **New Template**. A modal appears:

<!-- SCREENSHOT: new-template-form -->

Fill in:
- **Name** — human-readable template name (e.g., "My First Template")
- **Project** — which project to add this template to
- **Parent** — select a parent template for the hierarchy, or leave blank for a root template
- **Git path** — the file path within the project's `git_path` (e.g., `my_first_template.j2`)

Click **Create**.

---

## Step 3: The Template Editor

The template editor opens with two panels:

<!-- SCREENSHOT: template-editor-empty -->

- **Left panel** — Jinja2 template body (code editor)
- **Right panel** — Parameters, Data Sources, Features tabs

Write a minimal Jinja2 template body in the left panel:

```jinja2
hostname {{ device.hostname }}
description Generated by {{ glob.company_name }}
```

<!-- SCREENSHOT: template-editor-with-body -->

---

## Step 4: Add a Parameter

Click the **Parameters** tab in the right panel, then click **Add Parameter**.

<!-- SCREENSHOT: add-parameter-modal -->

Fill in:
- **Name** — `device.hostname`
- **Label** — "Device Hostname"
- **Widget** — `text`
- **Required** — ✅

Click **Save**.

The parameter appears in the parameters list with a drag handle to reorder it.

---

## Step 5: Save and Commit

Click **Save** (or **Commit to Git**). A commit message prompt appears. Enter a brief message describing the change (e.g., "Add my-first-template") and confirm.

The template file is written to Git and the DB record is updated with the new parameter.

---

## Step 6: Verify in the Catalog

Navigate to **Catalog**, open your project, and find the new template. Click it to open the render form.

<!-- SCREENSHOT: parameter-in-form -->

You should see the `device.hostname` field (labeled "Device Hostname") in the Template section of the form.

---

## Step 7: Test Render

Fill in `device.hostname` with a test value (e.g., `test-device-01`) and click **Render**.

<!-- SCREENSHOT: render-new-template -->

The output should contain:
```
hostname test-device-01
description Generated by <your org name>
```
